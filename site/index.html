--csstyle
clean simple styling for the web

csstyle is a modern approach for crafting beautifully maintainable, readable, refactorable stylesheets. The provided high level SASS mixins codify the csstyle method, rendering specificity and loading order irrelevant, and making your project refreshingly consistent

Components

With csstyle you organize your CSS as reusable components.

HTML 
Neat!
SCSS 
@include component(button) {
  background: #6BD9ED;
  padding: 15px;
}
RESULT
Neat!

Components are the primary building blocks of your project, the more you leverage them the better.

csstyle's focus on components make it a perfect match for Web/Angular/Ember/React components. Tip: create a separate file for each component to make them easy to find & edit.

Options

Make your components super flexible with options, passed as arguments to the component. Options override the component's default styling.

HTML 

  Register Now
SCSS 
@include component(button) {
  background: #6BD9ED;
  padding: 15px;

  @include option(action){
    color: #1F1F1F;
    background: #A7E040;
  }
}
RESULT
Register Now

Component options are a great way to organically grow your CSS over time. When you go to style something you can often just add a new option to an existing component. Code reuse FTW.

Parts

Components are composed of parts, child elements of the component that can be styled. Parts can also have their own options for even greater flexibility. Parts are applied using the component name followed by __ and the part name.

HTML 

  Register Now
SCSS 
@include component(button) {
  background: #6BD9ED;
  padding: 15px;

  @include option(action){
    color: #1F1F1F;
    background: #A7E040;
    padding-top: 0;
  }

  @include part(icon){
    font-size: 34px;
    position: relative;
    top: 10px;

    @include option(rocket){
      &:before {
        content: "\e600";
      }
    }
  }
}
RESULT
Register Now

Parts make it easy to break complex components up into manageable chunks.

Tweaks

Create generic style tweaks that can be applied to any component instance or element. Tweaks are applied using a + sign followed by the tweak name. Tweaks automatically override the styling of components, options, and parts.

HTML 

  Register Now
SCSS 
@include tweak(rounded) {
  border-radius: 10px;
}

@include tweak(glass) {
  box-shadow: inset 0 0 6px rgba(0, 0, 0, .8), 
              inset 0 9px 1px rgba(0, 0, 0, .1);
}
RESULT
Register Now

Tweaks further promote clean organization and style reuse. Refactor common/duplicate styles out of components and into tweaks. Tweaks should generally produce a single effect.

Locations

Projects occasionally need to override styles in specific locations e.g. just on the home page or just when viewing in fullscreen. Locations are applied using an @ sign followed by the location name. Locations automatically override the styling of components, options, parts and tweaks.

HTML 

  
    Register Now
SCSS 
@include location(home) {
  color: #6BD9ED;
  background: #1F1F1F;
}
RESULT
Register Now

Generally it is better to create new reusable component options, but sometimes location-specific overrides make a lot of sense.
